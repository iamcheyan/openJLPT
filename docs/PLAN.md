# OpenJLPT — 离线JLPT刷题App 项目计划

## 一、项目定位

一个**完全离线**的JLPT刷题App，目标运行在墨水屏设备上。

核心特点：
- 不需要网络
- 不需要登录
- 不需要API
- 白底黑字、无动画、低刷新 → 适合墨水屏

最终产品形态：一个"完全离线的JLPT刷题App"，而不是一个AI工具。

## 二、技术架构

```
┌─────────────────────────────────┐
│         Android App (壳)         │
│  ┌───────────────────────────┐  │
│  │        WebView             │  │
│  │  ┌─────────────────────┐  │  │
│  │  │   index.html         │  │  │
│  │  │   style.css          │  │  │
│  │  │   app.js             │  │  │
│  │  └────────┬──────────────┘  │  │
│  │           │ 读取             │  │
│  │  ┌────────▼──────────────┐  │  │
│  │  │   questions.json      │  │  │
│  │  │   (本地题库)           │  │  │
│  │  └───────────────────────┘  │  │
│  └───────────────────────────┘  │
└─────────────────────────────────┘
```

前端技术栈：纯HTML + CSS + Vanilla JS（不用框架，保持轻量）
存储：localStorage（错题、进度）
打包：WebView壳 → APK

## 三、开发步骤（按顺序）

---

### Step 1: 题库 — 确定题型和题库结构

这是整个项目的基础，必须先做好。

#### 1.1 JLPT考试结构

JLPT分为N5~N1五个级别，每个级别的考试都包含以下几大部分：

| 部分 | 日文名 | 说明 | N5题量参考 | N3题量参考 |
|------|--------|------|-----------|-----------|
| 文字語彙 | 文字・語彙 | 汉字读音、词汇选择 | ~25题 | ~35题 |
| 文法 | 文法 | 语法选择、句子排列 | ~20题 | ~30题 |
| 読解 | 読解 | 阅读理解 | ~15题 | ~25题 |
| 聴解 | 聴解 | 听力理解 | ~25题 | ~30题 |

> 注：聴解需要音频，第一版先不做，只做文字語彙 + 文法 + 読解。

#### 1.2 文字語彙 — 题型一览

**题型A：汉字选读音**
给一个包含汉字的句子，选正确的假名读音。
例：「この大学は有名です。」→ だいがく / たいがく / だいがくう / たいがくう

**题型B：读音选汉字**
给一个假名写的词，选正确的汉字。
例：「ひろい」→ 広い / 大きい / 長い / 遠い

**题型C：选词填空**
句子中有空格，从四个词里选一个填入。
例：「毎日（　）をします。」→ 運動 / 通勤 / 旅行 / 勉強

**题型D：同义词/近义词**
给一个词，选意思最接近的。
例：「たのしい」→ 楽しい / 嬉しい / 面白い / 美しい

#### 1.3 文法 — 题型一览

**题型A：语法选择**
句子中有空格，选正确的语法表达。
例：「雨が降った（　）、試合は中止になりました。」→ ので / のに / ところで / として

**题型B：句子排序**
给出打乱的词语片段，选出正确的排列顺序。
例：（までに）（提出して）（金曜日）（ください）→ 金曜日までに提出してください

**题型C：语法接续**
选词填入语法结构的空格。
例：「日本に来て（　）日本語が上手になりました。」→ から / まで / ので / のに

#### 1.4 読解 — 题型一览

**题型A：短文阅读**
一篇短文（3~5句），配1~2个问题。
例：一段关于周末活动的短文 → 问"いつ映画を見に行きましたか"

**题型B：中长文阅读**
一篇较长的文章（10~20句），配3~5个问题。
例：一段关于日本学校制度的文章 → 问多个细节问题

#### 1.5 需要讨论决定的问题

- [ ] 先做哪个级别？（建议N3，适用人群最广）
- [ ] 每个级别每种题型要做多少题？
- [ ] 题目来源：AI生成 + 人工校对？还是从已有题库导入？
- [ ] 解析用什么语言？（建议日文+中文混合）
- [ ] 第一版先做哪几种题型？（可以先做最核心的几种，后面再扩展）

### Step 2: 前端页面 — 做一个能刷题的静态网页

**目标：** 浏览器打开index.html就能刷题

功能清单：
- [ ] 选择级别（N5~N1）
- [ ] 选择部分（文字語彙 / 文法 / 読解）
- [ ] 显示题目和选项
- [ ] 点选答案，显示对错（绿色/红色）
- [ ] 显示解析
- [ ] 下一题 / 上一题
- [ ] 题目计数（第X题 / 共Y题）

文件结构：
```
web/
├── index.html          ← 入口页面
├── style.css           ← 样式（墨水屏友好）
├── app.js              ← 逻辑（读取JSON、判题、翻页）
└── data/               ← 题库（从Step1产出）
    └── N3/
        ├── vocabulary.json
        ├── grammar.json
        └── reading.json
```

---

### Step 3: 墨水屏适配 — 确保在墨水屏上体验好

**目标：** 白底黑字、无动画、低刷新

约束清单：
- [ ] 禁止所有CSS动画和过渡（`transition: none`, `animation: none`）
- [ ] 白底黑字，高对比度
- [ ] 字体足够大（建议正文18px+，选项20px+）
- [ ] 减少不必要的DOM更新（墨水屏全页刷新成本高）
- [ ] 避免滚动，尽量一屏显示完整内容
- [ ] 按钮要大，方便触控操作
- [ ] 测试：在浏览器里模拟（关动画、调大字体）

---

### Step 4: 错题本 — 记录做错的题目

**目标：** 做错的题自动记录，可以单独复习

功能：
- [ ] 答错自动存入localStorage
- [ ] 错题列表页面（显示题目、错误次数）
- [ ] 从错题本进入重新做题
- [ ] 答对后可以从错题本移除
- [ ] 清空错题本功能

localStorage结构：
```json
{
  "wrongAnswers": {
    "n3-vocab-001": { "count": 3, "lastWrong": "2026-05-03" },
    "n3-gram-005": { "count": 1, "lastWrong": "2026-05-02" }
  }
}
```

### Step 5: 进度统计 — 记录刷题进度

**目标：** 知道自己刷了多少、正确率多少

功能：
- [ ] 总做题数 / 总题数
- [ ] 正确率（整体 + 按级别 + 按分类）
- [ ] 每日做题记录
- [ ] 数据存在localStorage

### Step 6: 每日刷题计划（可选扩展）

**目标：** 设定每天刷多少题，按计划推送

功能：
- [ ] 设定每日目标（如每天20题）
- [ ] 自动从题库中未做的题目中选题

### Step 7: Android WebView壳 — 打包成APK

**目标：** 把静态网页包进Android App，安装到设备上

结构：
```
android/
├── app/
│   ├── src/main/
│   │   ├── java/.../MainActivity.java
│   │   ├── assets/
│   │   │   └── web/          ← 把Step2的web目录放这里
│   │   │       ├── index.html
│   │   │       ├── style.css
│   │   │       ├── app.js
│   │   │       └── data/
│   │   │           └── N3/
│   │   │               ├── vocabulary.json
│   │   │               ├── grammar.json
│   │   │               └── reading.json
│   │   └── res/
│   └── build.gradle
└── build.gradle
```

核心代码（MainActivity.java）大概只需要：
```java
WebView webView = findViewById(R.id.webview);
webView.getSettings().setJavaScriptEnabled(true);
webView.loadUrl("file:///android_asset/web/index.html");
```

需要决定：
- [ ] 最低Android版本
- [ ] 是否需要横竖屏锁定
- [ ] App名称和图标
