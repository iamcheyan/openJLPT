# JLPT N2 题库 JSON 结构定义

## 文件组织

```
data/
└── n2/
    ├── vocab_reading.json      ← ① 漢字の読み方
    ├── vocab_kanji.json        ← ② 漢字の表記
    ├── vocab_context.json      ← ③ 文脈規定
    ├── vocab_synonym.json      ← ④ 言い換え・類義
    ├── vocab_usage.json        ← ⑤ 用法
    ├── grammar_fill.json       ← ⑥ 文の文法
    ├── grammar_order.json      ← ⑦ 文の組み立て
    ├── grammar_passage.json    ← ⑧ 文章の文法
    ├── reading_short.json      ← ⑨ 短文読解
    ├── reading_medium.json     ← ⑩ 中文読解
    ├── reading_long.json       ← ⑪ 長文読解
    └── reading_search.json     ← ⑫ 情報検索
```

---

## 通用字段说明

| 字段 | 类型 | 必须 | 说明 |
|------|------|------|------|
| id | string | 是 | 唯一标识，格式：`n2-{type}-{序号}` |
| level | string | 是 | 固定 `"N2"` |
| question_type | string | 是 | 题型标识（见下方） |
| source | string | 否 | 出题来源（`"ai"` 或 `"official"`） |
| generated_by | string | 是 | 出题模型名，如 `"ark:doubao-seed-2-0-pro"` |
| reviewed_by | string | 否 | 审题模型名，如 `"zhipu:glm-4-flash"` |
| verified | boolean | 是 | 是否经过审核通过 |
| review_explanation | string | 否 | 审题模型给出的详细中文解析（见下方） |
| created_at | string | 否 | 创建时间 ISO8601 |

## review_explanation 字段规范

审题模型必须给出详细的中文解析，包含以下内容：

1. **答案解释** — 为什么这个选项是正确的
2. **干扰项分析** — 其他选项为什么错误
3. **语法/词汇重点** — 这道题考的核心知识点
4. **学习建议** — 用户做错了应该记住什么

示例：
```
「貴重」读作「きちょう」（二声），意思是珍贵、贵重。注意「貴」在这里读「き」而不是「きい」。
干扰项分析：
- きじゅう：常见误读，把「貴」读成了「きい」的浊音化
- きっじゅう：促音错误，「貴」不产生促音
- きっちょう：这是另一个词「几帳面」的读音，与本题无关
语法重点：汉字词的音读规律，特别是单音节汉字（一字一音）的读法
学习建议：「貴重」是N2高频词，建议记住「きちょう」这个读音，常见搭配有「貴重な体験」「貴重品」
```
